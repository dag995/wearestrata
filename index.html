The problem

The contract says one thing. The system charges another.

Every organisation that pays or collects charges governed by a contract faces the same structural problem. The contract contains the rules. The systems contain the transactions. Nothing connects them.

Sellers operationalise contracts
Service providers know exactly which clauses permit emergency rates, after-hours premiums, and material markups. Their billing systems are configured to capture every entitlement. This is rational commercial behaviour.
Buyers file contracts
Organisations negotiate hard upfront then put the document in a drawer. Invoices are processed against PO amounts. Nobody checks whether each line complies with the actual contract terms because the cost of doing so exceeds the value of any individual discrepancy.
Leakage accumulates silently
Across thousands of invoices, the discrepancies compound. Charges applied without clause authority. Entitlements not collected. Apportionments based on precedent rather than the current contract. The errors are not visible until someone reads the contract. Nobody reads the contract.
Manual review does not scale
A forensic auditor costs £50 to £150 per review. That is rational for a sample, not a portfolio. The problems the auditor finds in the sample exist everywhere. They reappear in the next billing cycle because the underlying process has not changed.

How it works

Extract. Confirm. Freeze. Validate everything.

Strata treats contract compliance as a caching problem, not a repeated inference problem. Machine learning and large language models extract the rules. Your domain experts confirm them. Once confirmed, the rules are frozen into a deterministic logic graph and applied to every charge, identically, forever. No hallucination. No drift. No interpretation variability.

1
Extract the rules
Every contract, lease, policy, schedule, and regulatory instrument analysed using a combination of machine learning extraction and large language model interpretation. ML models identify clause boundaries, classify charge types, and extract structured entities. LLMs interpret conditional logic, cross-references, and commercial intent. The output is not a summary. It is a structured set of executable rules: pricing models, scope boundaries, eligibility criteria, apportionment methodologies, cap provisions, escalation conditions, penalty thresholds. Each rule linked to its source clause.
2
Confirm with your experts
Extracted rules are presented to your domain experts for review and confirmation. Every rule is shown alongside the source clause it was derived from. Your team confirms, adjusts, or rejects each rule. This is the step that eliminates hallucination risk entirely. No rule enters the compliance layer without human confirmation. Once confirmed, the rule is frozen: deterministic, auditable, and applied identically from that point forward.
3
Cache the confirmed rules
Confirmed rules are stored in a patent-pending three-tier memory hierarchy optimised for the access patterns of compliance workloads. Rules are indexed across five dimensions: parameter, entity, time period, dependency, and semantic similarity. Any query resolves instantly against the frozen logic graph. No AI is involved in the validation step. The rules are deterministic code, not probabilistic inference.
4
Validate every charge
Every invoice line, service charge, cost allocation, and rechargeable item matched against the cached rules. Is this charge within scope? Is this rate current? Is this apportionment correct? Does this clause permit this cost category? Every deviation flagged with the specific clause reference, the reasoning, and the financial impact.
5
Update incrementally
When a contract is amended, Strata detects which sections have changed, re-extracts only the affected rules, and invalidates dependent logic. No full re-read. No manual reconfiguration. The cached knowledge stays current automatically. Version history enables retrieval of rules as they were at any past date.
✓
Prove it
Every validated charge linked to the source rule, the source clause in the contract, and the input data used in the computation. A complete, immutable, deterministic audit trail. The same rules and the same data produce identical results on every execution. Auditor-ready. Tribunal-ready. Board-ready.

Recursive Logic Caching

A world-first architecture that combines machine learning extraction, large language model interpretation, and expert confirmation to compile unstructured contract text into a deterministic, frozen logic graph. Other approaches force a choice: the hallucination risks of generative AI, or the manual coding bottleneck of rules engines. Strata eliminates the trade-off. ML and LLMs do the extraction. Your experts confirm the rules. The confirmed rules execute deterministically, with zero AI involvement in the validation step.

Three-tier memory hierarchy
Hot, warm and cold cache tiers optimised for the access patterns of compliance workloads. The most frequently accessed rules are served instantly. The architecture scales across portfolios of any size without degradation.
Probabilistic pre-fetch
Statistical co-occurrence models learn which rules are queried together. Related rules are pre-loaded into the hot cache before they are needed. 76% of queries benefit. 31x effective speedup from pre-fetching.
Five-dimensional indexing
Rules indexed by parameter, entity, time period, dependency graph, and semantic vector. Queries decomposed into multi-index lookups. O(log R) retrieval complexity regardless of total rule volume.
Incremental update controller
Cryptographic hashing detects which contract sections have changed. Only affected rules are re-extracted. Dependent rules are invalidated and recomputed. No full re-read required. Full version history preserved for any point-in-time audit.
Deterministic execution
AI is used for extraction. It is not used for validation. Confirmed rules are executable logic, not probabilistic text. Given the same rules and the same input data, the engine produces identical output on every execution. Zero hallucination risk. Complete reproducibility for audit and dispute. The separation between AI-powered extraction and deterministic execution is fundamental to the architecture.
Document agnostic
The extraction and caching architecture does not assume a document type. Leases, contracts, policies, regulations, framework agreements, licence terms. Any unstructured text containing commercial logic can be compiled into the graph.
